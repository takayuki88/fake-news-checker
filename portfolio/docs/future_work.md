# 今後の改善

## 1. blind testの作成

既存100件は改善過程でも確認に使った評価データです。
次は改善に使っていない新規データ100件を作成し、未知データにも通用するかを確認します。

## 2. バージョン比較

Ver2 / Ver3 / Ver4を同じblind setで比較します。
同じ条件で比較することで、どの改善が実際に効果を持ったのかを確認します。

## 3. 境界判定ルールの構造化

現在の課題は、「正確」「ほぼ正確」「不正確」の境界です。
固有名詞ごとの個別対応ではなく、数字、時期、人物名、主張の中心との関係から一般化できるルールを整えます。

## 4. 実運用に向けた改善

- 処理時間の短縮
- 根拠リンクの見やすさ改善
- UIの改善
- 複数記事の一括処理
- 判定履歴の記録化
- レポート出力

## 5. 目指す状態

最終的には、広報、危機管理、SNS監視などの現場で、大量情報の一次スクリーニングとして使える形を目指します。
AIによる断定ではなく、人間が確認するための材料を速く、説明可能な形でそろえることを重視します。
