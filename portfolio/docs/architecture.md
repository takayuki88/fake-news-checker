# システム構成

## 目的

Fake News Checkerは、ニュース記事やSNS投稿の真偽確認を支援するためのアプリケーションです。
最終判断をAIに任せるのではなく、人間が確認すべき情報を整理し、判定理由と根拠を提示することを目的にしています。

## 処理の流れ

```text
入力
  ↓
本文抽出、メタ情報整理
  ↓
ローカル一次判定
  ↓
GPT一次レビュー
  ↓
Gemini外部根拠確認
  ↓
補正ルール
  ↓
5段階判定、理由、根拠リンクを表示
```

## 各ステップの役割

| ステップ | 役割 |
| --- | --- |
| 本文抽出 | URL入力時に記事本文、著者、公開日時などを取得する |
| ローカル一次判定 | 最低限のヒューリスティックで補助判定を行う |
| GPT一次レビュー | 文脈、論点、主張の中心を整理する |
| Gemini外部根拠確認 | Google Searchを使い、外部根拠と主張を比較する |
| 補正ルール | LLM出力を5段階ラベルへ安定して変換する |

## 判定ラベル

| ラベル | 意味 |
| --- | --- |
| 正確 | 主張が根拠と一致している |
| ほぼ正確 | 大筋は正しいが、細部に補足や軽微な差分がある |
| 判断保留 | 根拠不足などにより断定できない |
| 不正確 | 一部または重要部分に誤りがある |
| 誤り | 主張の中心が根拠と明確に矛盾している |

## 設計上の工夫

- 本文の自然さだけで真偽を決めない
- LLMの出力をそのまま採用せず、外部根拠確認と補正ルールを組み合わせる
- 二値分類ではなく5段階ラベルで曖昧さを扱う
- APIキー未設定時もローカル判定で動くようにする
- 評価データを使って、改善結果を数値で確認できるようにする
